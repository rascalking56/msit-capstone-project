# MSIT 5910 Capstone Project

## 📘 Overview
This repository contains the source code, documentation, and design artifacts for the MSIT 5910 Capstone Project at the University of the People.  
The project demonstrates the design and implementation of a modular, scalable, and secure information system architecture.

---

## 🧩 System Architecture
The system follows a **layered architecture** consisting of:
- **Presentation Layer:** User interface components for data input and visualization.
- **Application Layer:** Core logic and workflow orchestration.
- **Data Layer:** Database and storage management.
- **Integration Layer:** External APIs and authentication services.

Refer to `/design/architecture-diagrams` for DFD, UML, and ERD diagrams.

---

## ⚙️ Functional Requirements (FRs)
1. Secure user authentication and authorization  
2. Data capture and validation  
3. Workflow execution and result generation  
4. CRUD operations for persistent data  
5. Integration with external APIs  

### Non‑Functional Requirements (NFRs)
- **Performance:** Fast response times (<200 ms for core operations)  
- **Usability:** Intuitive and accessible interface  
- **Reliability:** Fault‑tolerant workflows and backups  
- **Scalability:** Horizontal scaling and modular design  
- **Security:** Encryption, RBAC, and audit logging
